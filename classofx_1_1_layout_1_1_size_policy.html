The box can grow beyond its size hint if necessary.

The box should get as much space as possible.

enum ofx::Layout::SizePolicy::Type
Enumerator
FIXED 

The sizeHint() is the only acceptable alternative, so the box can never grow or shrink (e.g. the vertical direction of a push button).

MINIMUM 

The sizeHint() is minimal, and sufficient. The box can be expanded, but there is no advantage to it being larger (e.g. the horizontal direction of a push button). It cannot be smaller than the size provided by sizeHint().

MAXIMUM 

The sizeHint() is a maximum. The box can be shrunk any amount without detriment if other boxes need the space (e.g. a separator line). It cannot be larger than the size provided by sizeHint().

PREFERRED 

The sizeHint() is best, but the box can be shrunk and still be useful. The box can be expanded, but there is no advantage to it being larger than sizeHint() (the default Box policy).

EXPANDING 

The sizeHint() is a sensible size, but the box can be shrunk and still be useful. The box can make use of extra space, so it should get as much space as possible (e.g. the horizontal direction of a horizontal slider).

MINIMUM_EXPANDING 

The sizeHint() is minimal, and sufficient. The box can make use of extra space, so it should get as much space as possible (e.g. the horizontal direction of a horizontal slider).

IGNORED 

The sizeHint() is ignored. The box will get as much space as possible.
